# 🚀 Projetos

Guia para escolher, registrar e concluir projetos ao longo do curso de ADS.

---

## 🎯 Como Escolher um Projeto por Série

| Série | Foco recomendado | Exemplos |
|---|---|---|
| 1ª | Fundamentos e prototipação | Simulador de rede, protótipo de interface |
| 2ª | Algoritmos e modelagem | CRUD básico, visualizador de algoritmos |
| 3ª | Orientação a objetos e dados | Sistema com design patterns, mini-ORM |
| 4ª | Web e nuvem | API REST, SPA com framework, deploy na nuvem |
| 5ª | Qualidade, mobile e inovação | App mobile, suite de testes, projeto integrador |

**Critérios para escolha:**
- Está alinhado com pelo menos uma disciplina da série atual?
- É possível concluir uma versão mínima em 2-4 semanas?
- Agrega ao portfólio técnico?

---

## 🛠️ Stack (a definir por projeto)

A linguagem e o framework serão escolhidos conforme o projeto e a série. Sugestões:

| Camada | Opções |
|---|---|
| Back-end | Python, Java, Node.js |
| Front-end | HTML/CSS/JS, React, Vue |
| Banco de dados | SQLite, PostgreSQL, MySQL |
| Mobile | React Native, Flutter |
| Testes | Jest, JUnit, Pytest |
| Nuvem | AWS Free Tier, Railway, Render |

---

## ✅ Critérios de "Feito"

Um projeto é considerado **concluído** quando:

- [ ] Possui um `README.md` próprio com:
  - Descrição do projeto
  - Tecnologias utilizadas
  - Instruções de instalação e execução
  - Link para série/disciplina relacionada
- [ ] Tem pelo menos **testes básicos** documentados ou implementados
- [ ] O código está organizado e comentado onde necessário
- [ ] Screenshots ou demos estão em [`assets/`](../assets/)
- [ ] Está listado neste README com link para a pasta
