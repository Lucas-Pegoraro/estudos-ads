# 📚 Estudos ADS — Análise e Desenvolvimento de Sistemas

Repositório público para organizar meus estudos do curso de **Análise e Desenvolvimento de Sistemas (ADS)**.  
Serve como caderno digital + portfólio, reunindo anotações autorais, exercícios resolvidos e pequenos projetos.

---

## 🎯 Objetivo

- Consolidar o aprendizado de cada disciplina por semestre.
- Criar um portfólio técnico progressivo ao longo do curso.
- Facilitar revisão e consulta rápida de conteúdos.

---

**Convenção de commits:**
```
feat(disciplina): adiciona anotacoes sobre [tema]
fix(exercicio): corrige solucao do exercicio 03
docs(serie): atualiza checklist da 2a serie
```

## ✅ Boas Práticas

### Como escrever commits

Use a convenção [Conventional Commits](https://www.conventionalcommits.org/):

```
<tipo>(escopo): descrição curta em minúsculas

Exemplos:
feat(redes): adiciona anotacoes sobre modelo OSI
fix(exercicio): corrige logica do exercicio 05 de algoritmos
docs(readme): atualiza checklist da 3a serie
refactor(codigo): reorganiza exemplos de POO
```

### Como registrar dúvidas e pendências em issues

1. Acesse a aba **Issues** do repositório.
2. Clique em **New Issue**.
3. Use o título no formato: `[Disciplina] Dúvida sobre <tema>`.
4. Descreva a dúvida com contexto suficiente para você mesmo entender depois.
5. Adicione labels como `duvida`, `pendente`, `revisao` conforme necessário.
6. Feche a issue quando a dúvida for resolvida, adicionando um comentário com a resposta.
